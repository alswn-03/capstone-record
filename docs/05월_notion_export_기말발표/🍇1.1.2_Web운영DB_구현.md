# 2. 더미 데이터 및 SQL 코드 (1학기기말ver)

## 1-1. table 생성

```bash
-- items 테이블 생성
CREATE TABLE IF NOT EXISTS items (
    id VARCHAR(255) PRIMARY KEY,
    item_name VARCHAR(255) NOT NULL,
    price INT NOT NULL,
    information TEXT,
    image_url VARCHAR(255)
);

-- users 테이블
CREATE TABLE IF NOT EXISTS users (
    id VARCHAR(255) PRIMARY KEY,
    user_name VARCHAR(255) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);

-- transactions 테이블
CREATE TABLE IF NOT EXISTS transactions (
    id INT AUTO_INCREMENT PRIMARY KEY,
    user_id VARCHAR(255) NOT NULL,
    item_id VARCHAR(255) NOT NULL,
    timestamp DATETIME DEFAULT CURRENT_TIMESTAMP,
    FOREIGN KEY (user_id) REFERENCES users(id),
    FOREIGN KEY (item_id) REFERENCES items(id)
);
```

## 🤔 image_url 어디에 저장할 것인가? (작업 플로우 계획)

- 일단은

    1. `client/public/images`이미지 저장 폴더 만들고 → 여기에 이미지(png) 저장
        - 경로: `client/public/images/item_01.png`

    2. DB에는 image_url을 text로 저장하고, 화면 띄울 때는 프론트엔드에서 불러온다

    - DB 저장 값: `/images/item_01.png`
    - DB에서 가져온 `image_url` 값을 프론트엔드의 `src`에 넣어주기

> 
> 왜? 연결이 잘 되는지 파이프라인들을 확인해보는 게 먼저니까, 일단은 로컬 폴더를 사용하고,
> 
> 나중에 진짜 추천 모델을 돌려야 해서/병목을 찾아내야해서, 관리할 item이 많아졌을 때, S3(클라우드 스토리지)로 이동(migration)시키자
> 


- 나중에 S3로 전환시킬 때는 

    - **이미지:** `client/public/images` 폴더에 있던 사진 파일들을 그대로 AWS S3 버킷에 업로드
    - **DB 데이터 수정:** DB의 `items` 테이블에 저장된 `image_url` 값들을 수정.
    기존의 `/images/item_01.png`를 S3가 제공하는 절대 주소(예: `https://my-bucket.s3.ap-northeast-2.amazonaws.com/item_01.png`)로 업데이트(`UPDATE`)만 해주면 됩니다.
    - **프론트엔드 코드는 수정 없음!:** 프론트엔드 코드는 이미 `<img src={item.image_url} />` 형태로 변수를 받아오도록 작성되어 있기 때문에, **단 한 줄의 코드도 수정할 필요가 없습니다.** 브라우저가 변경된 S3 주소를 읽고 알아서 이미지를 띄워줍니다.

    cf) 나중에 '관리자용 상품 등록 페이지'가 필요해진다면, 그때 가서 백엔드(`server`)에 S3 업로드용 라이브러리(`multer-s3` 등)를 추가해서 API를 만들어주기만 하면 완벽하게 확장이 가능
