# 1.1.1 웹에서 이벤트 로그는 어떻게 생성되는가?

상태: 완료

> 로그 이벤트 생성은 어떻게 되는건지. 보통 어떻게 생성?시키는지.
아니면 원래도 생성되는데 완전 raw하게, 이걸 기업들이 가공해서 log event 로써 만드는 건지. 
아니면 코드 ? 문법 같은 장치가 있는건지.
> 

<aside>

**프론트엔드**에서 사용자의 행동을 감지하고, 이를 **백엔드(데이터 수집 서버)**로 **전송(API 호출)하는 과정**을 거침

</aside>

- 단순하게 console.log 를 찍는 것은 아님.
    
    console.log는 사용자의 브라우저(클라이언트)에만 남기 때문에 데이터를 수집할 수 없음
    

<aside>

### **ex) 구현 과정 : 프론트엔드 + 백엔드 수집 API**

---

1. **이벤트 리스너 부착 :** 
    
    view(상품 클릭), addtocart, transaction(결제) 버튼에 클릭 이벤트를 달아둔다
    
2. **JSON 데이터 구성 :** 
    
    클릭 발생하면, 설계한 스키마(event_time, user_id, item_id 등)에 맞게 JSON 객체를 만든다
    
3. **서버로 전송 :** 
    
    이 JSON 데이터를 백엔드의 수집 API로 보낸다
    
    - 보통 `fetch`, axios 또는 브라우저 종료 시에도 안전하게 보내주는 navigator.sendBeacon을 사용
- 프론트엔드 코드 예시(javascript)
    
    ```jsx
    // 장바구니 버튼 클릭 시 실행되는 함수
    function handleAddToCart(itemId, userId) {
      const eventLog = {
        event_time: Date.now(), // unix timestamp (ms)
        user_id: userId,
        item_id: itemId,
        event_type: "addtocart",
        session_id: getSessionId() // 로컬 스토리지 등에서 세션 ID를 가져옴
      };
    
      // 백엔드 로그 수집 API로 데이터 전송
      fetch("https://api.yourdomain.com/collect-log", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(eventLog)
      });
    }
    ```
    
</aside>

#### 실제 기업에서도 위 구현 방법을 사용하는가? YES

- **SDK(Software Development Kit) 란,**
    
    자주 쓰는 복잡한 기능들을 누구나 쉽게 가져다 쓸 수 있도록 하나로 묶어놓은 코드 패키지
    
    - 밀키트(Meal Kit) 같은 거임. 요리할 때 재료 일일이 다 씻고, 다지고 하지 않고, 다 준비된 밀키트 뜯어서 볶기만 하면 되는거니까 (=SKD사용)
    - 회사에서 로깅 시스템을 구축할 때 사내 SDK를 만든다는 것은
        - 1. SDK가 없을 때 (원시적인 방법)
            
            상품 클릭 로그를 수집하려면, 상품 상세 페이지, 장바구니 페이지, 결제 페이지 등 모든 프론트엔드 코드마다 
            
            백엔드 서버 주소를 적고, `fetch` 함수를 부르고, 에러가 나면 어떻게 할지 길고 복잡한 코드를 매번 중복해서 작성해야 함
            
        - 2. SDK를 만들어서 쓸 때 (실무 방식)
            
            데이터 엔지니어나 프론트엔드 리드 개발자가 이 복잡한 통신 과정, 유저 ID 수집, 에러 처리 로직을 모두 하나로 묶어서 `Logger.track()`이라는 간단한 함수(SDK)로 만들어 사내에 배포
            그러면 다른 프론트엔드 개발자들은 복잡한 `fetch` 과정은 몰라도, 그냥 버튼 클릭 코드 밑에 `Logger.track('addtocart')` 한 줄만 적으면 완벽하게 로그가 전송됨
